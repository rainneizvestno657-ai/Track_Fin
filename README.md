# Track_Fin — Nordic Dark Minimalist Finance Tracker

Десктопное приложение для учёта личных финансов на Python + CustomTkinter.  
Тема оформления: **Nordic Dark Minimalist**.

---

## Структура проекта

```
Track_Fin/
│
├── main.py                  # Точка входа. WindowManager: окно, sidebar, навигация
│
├── core/                    # Слой данных и бизнес-логики
│   ├── __init__.py
│   ├── data_manager.py      # Состояние: баланс, транзакции, категории, цели, экспорт
│   ├── locale_manager.py    # i18n: загрузка JSON-переводов, t(), month_suffix()
│   └── utils.py             # Вспомогательные функции (format_kgs)
│
├── locales/                 # Файлы переводов
│   ├── ru.json              # Русский
│   ├── en.json              # English
│   └── ky.json              # Кыргызча
│
├── views/                   # Слой представления (по одному файлу на экран)
│   ├── __init__.py
│   ├── transactions_view.py # Дашборд транзакций
│   ├── analytics_view.py    # Аналитика и сравнение
│   ├── categories_view.py   # Категории с лимитами
│   ├── goals_view.py        # Цели накопления
│   ├── settings_view.py     # Настройки языка и экспорт данных
│   └── help_view.py         # FAQ и о приложении
│
├── requirements.txt
└── .gitignore
```

---

## Архитектурные принципы

| Принцип | Реализация |
|---|---|
| Разделение ответственности | `core/` — данные и логика; `views/` — только UI |
| Единственная ответственность | Каждый модуль отвечает за одну задачу |
| Инверсия зависимостей | View-файлы получают `controller` через конструктор, не импортируют глобальные объекты |
| i18n через JSON | Все строки UI в `locales/*.json`, нет хардкода |

## Запуск

```bash
# 1. Создать и активировать виртуальное окружение
python -m venv .venv
.venv\Scripts\activate        # Windows
source .venv/bin/activate     # macOS / Linux

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Запустить
python main.py
```

## Зависимости

| Пакет | Назначение |
|---|---|
| `customtkinter >= 5.2` | Современные виджеты поверх tkinter |
| `pillow >= 10.0` | Обработка изображений (требуется CTk) |
| `openpyxl >= 3.1` | Экспорт в .xlsx (необязательно, fallback в CSV) |
